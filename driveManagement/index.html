DriveManager — Setup Guide
Deploy in 5 minutes · Everything you need below
What this does: You deploy a small Google Apps Script as a Web App on your Google account. This HTML file calls it to read/write your Google Drive. Your URL + password stay in your browser only.
1
Go to Google Apps Script
Open script.google.com and click New Project. Rename it to DriveManager.
2
Add Code.gs
In the editor you'll see Code.gs. Click the 💻 Code.gs tab above, copy the entire code, then replace everything in the editor with it.
3
Set your username & password in the code
At the top of Code.gs find these two lines and change them to your credentials:

var AUTH_USER = 'admin';
var AUTH_PASS = 'your_password';

Important Use a strong password — this protects access to your Drive.
4
Deploy as Web App
Click Deploy → New deployment
• Type: Web App
• Execute as: Me
• Who has access: Anyone (auth is handled by your password)

Click Deploy and authorize when prompted.
5
Copy the Web App URL
After deploying, copy the URL that looks like:
https://script.google.com/macros/s/AKfy.../exec

Paste it into the Web App URL field on the login screen, enter your username & password, and click Connect.
6
Optional: Set a Root Folder
By default the app shows your entire My Drive. To restrict it to one folder, copy the folder ID from its Drive URL and paste it into the ROOT_FOLDER_ID variable in Code.gs, then redeploy.
⚠️ After any code change you must create a new deployment (not update existing) to pick up changes. Keep track of your Web App URL — it changes per deployment.
Code.gs — Full Apps Script
Copy this entire block into your Apps Script project
After pasting, change AUTH_USER and AUTH_PASS at the top of the file, then deploy as Web App → Execute as: Me → Access: Anyone.
Q
Is my data secure?
Yes. The Apps Script runs as your Google account and requires your username/password for every request. The URL + credentials are saved only in your browser's localStorage. No third-party server is involved.
Q
Why "Access: Anyone" in the deploy settings?
The Web App needs to be reachable from your browser (this HTML file). Authentication is handled by your password inside the script — setting access to "Anyone" just means Google doesn't add its own OAuth layer on top.
Q
Can I host this HTML file somewhere?
Yes. Upload the HTML file to GitHub Pages, Netlify, or any static host. It will call your Apps Script Web App URL from anywhere.
Q
Uploads are failing — why?
Large files may exceed Apps Script's 6-minute timeout. For files above ~10 MB, consider uploading directly to Drive and using this app for organization only.
Q
I changed the code but nothing changed?
Apps Script caches old deployments. You must create a new deployment (Deploy → New deployment) every time you change Code.gs. Update existing deployment does NOT update live code.
